Wide Vectors
This is a program that traces the path of a charged particle in a customizable electric and magnetic field.
    Instructions:
  • E and B can be entered as functions of the particle's coordinates: x, y, and z
  • The path is projected onto the xy plane, with the origin at the lower left corner.
  • Proper syntax: use * for multiplication, and pow(a,b) for ab.
    Most basic math functions are included: exp(), sqrt(), sin(), asin() etc. as well as constants pi and e.
    Shorthand conditional operator ? : can make piecewise functions.
    The dist() function gives the distance between two points in space (it needs 6 arguments).
  • You can set the initial position and velocity of the particle by clicking and dragging on the screen, or by entering values in the sidebar.
  • When animation is enabled, the particle's velocity (blue) and acceleration (red) vectors are shown.
    "timescale" means how many time units equal one second.

How it works: The E and B fields give the particle's acceleration as a function of position and velocity. This makes a system of three second order differential equations, which can be turned into six first-order ones and solved (approximated) numerically given starting conditions. The computer makes a series of steps; at each step it evaluates the derivative function to determine where it should take the next step. The method used in this program is called the Runge Kutta Fourth Order Method. The neat thing about this method is that when the step size is divided by n, the error (difference between approximate and actual) is divided by n4, hence Fourth Order. For example, going from stepsize = 0.01 to 0.001 makes the error 10000 times less. Caveat, this only works if the function is sufficiently smooth. Also, javascript isn't very good at dealing with floats so the error will shoot up if you try using step sizes smaller than 10-5 (at that point the computation will probably crash your browser anyway).
